\doxysubsection{Detailed Description}
\mbox{\hyperlink{classClass}{Class}} for working with local application settings, stored in filesystem. 

\mbox{\hyperlink{classProvides}{Provides}} access to reading and writing application settings to local .ini file. Automatically load default settings on fresh application install. \mbox{\hyperlink{classClass}{Class}} mainly used as wrapper over {\bfseries{QSettings}} instance, providing useful invokable functions and properties avialable in QML.

\mbox{\hyperlink{classConfig_1_1Config}{Config}} class also provides useful macro for accessing it\textquotesingle{}s properties from different classes and namespaces\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{CONFIG(property\_name);}

\end{DoxyCode}
 Example usage of macro\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Network::startTelemetrySocket(CONFIG(\mbox{\hyperlink{classConfig_1_1Config_a29ca03d2272c4051d7c324cd6bac3b0b}{remoteIP}})\ +\ \textcolor{stringliteral}{"{}:"{}}\ +\ CONFIG(\mbox{\hyperlink{classConfig_1_1Config_a9400347615d36fd3eed76e4f027433df}{telemetryPort}}),\ CONFIG(\mbox{\hyperlink{classConfig_1_1Config_a752dbeb4be188be8d6e370263c380827}{telemetryFrequency}}));}

\end{DoxyCode}


In QML, exactly same code can be achieved using following example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Network.startTelemetrySocket(Config.remoteIP\ +\ "{}:"{}\ +\ Config.telemetryPort,\ Config.telemetryFrequency);}

\end{DoxyCode}
